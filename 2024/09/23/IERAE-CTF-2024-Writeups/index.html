我寫出來的題目主要是 pwn 和 rev，以及一題 misc。

pwn

This is warmup

Recon

題目有給 source code，觸發 SIGSEGV 就能拿到 FLAG。

有機會觸發 SIGSEGV 的地方在:

  • 整個程式碼中只有這一段會寫 memory
  • ncol 和 nrow 是使用者輸入的變數

write matrix 前面的程式碼會對 ncol 和 nrow 做一些檢查:

  • 看起來是防止使用者 integer overflow
  • 不讓 malloc 分配超過可用的 memory

所以目標是想辦法繞過檢查，並讓後面 write matrix 超過 malloc 的範圍，導致 segmentation fault。

Solution

要讓 write matrix 成功有 OOB write，需要 nrow * ncol 實際上大於 malloc 的大小。

那麼 nrow * ncol 一定要 overflow，讓 malloc 分配很小的 memory。

unsigned long long int = 2 ** 64 - 1，所以 nrow * ncol = 2 ** 64 + num, num < malloc 最大上限，num 是 malloc 實際分配的大小。

要繞過第一個條件判斷 if (nrow * ncol < nrow) 還需要 num < nrow。

滿足上面列出的兩個條件:

  • nrow * ncol = 2 ** 64 + num, num < malloc 最大上限
  • num < nrow
    就能成功 OOB write，導致 segmentation fault。

舉個例子，我挑 num = 80000，所以 nrow * ncol = 18446744073709631616。

將這個數分割成兩數當 nrow 和 ncol，nrow = 65536, ncol = 281474976710657，這就是一組可要造成 segmentation fault 的 input。

FLAG: IERAE{s33?n07_41w4y5_1_cr3a73_d1ff1cu1t_pr0b13m5}

copy-and-paste

Recon

題目有建議使用提供的 docker 環境，所以先把 docker 環境開好。

FLAG 取得條件和 This is warmup 一樣是觸發 SIGSEGV。

觀察 source code，copy() 的部分是將一個 buffer 複製到另一個 buffer，有機會造成 OOB write:

  • 但會檢查 buffer size，不讓大的 buffer 可以直接蓋到小的 buffer 上面。

要繞過這個檢查我第一個想法是想辦法讓 buf_size 不是實際上的 size。

往前翻找到 create_new_buf() 中 assign buf_size 的地方:

  • ftell 決定 buf_size

很幸運的 google pwn ftell 找到 HITCON 2017 其中一題的 Writeup

According to code above, we know the file size actually depends on the return value of fseek and ftell. if the FILE* pointer passed into fseek and ftell is point to a FILE struct of “/dev/stdin”, both fseek and ftell will fail and return -1. Things become more interesting, since file_size is an unsigned value, -1 is treated as a extremely large value(i.e. MAX_ULONGINT).

只要想辦法讓 fseek, ftell return error 就可以讓 size 為 -1，導致 overflow 使 size 變得超大，就可以繞過檢查，讓大的 buffer 蓋到看起來很大實際上比較小的 buffer 上，導致 segmentation fault。

Solution

fseek(3) — Linux manual page

ESPIPE The file descriptor underlying stream is not seekable (e.g., it refers to a pipe, FIFO, or socket).

The functions fgetpos(), fseek(), fsetpos(), and ftell() may also fail and set errno for any of the errors specified for the routines fflush(3), fstat(2), lseek(2), and malloc(3).

那麼只要去 container 中找到 pipe, FIFO, socket 的檔案，然後用他們 create buffer 就能成功取得 size = -1 的 buffer。

最後我是選擇 /var/run 和 /bin/bash，然後 /bin/bash 覆蓋到 /var/run。

FLAG: IERAE{7h3_f1rs7_s73p_7o_b3_4_pwn3r_51a7806b}
