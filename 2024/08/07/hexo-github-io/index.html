紀錄一下架設網頁的過程與遇到的問題。

前置作業

安裝 Node.js

後面需要用到 npm 安裝 Hexo，所以需要先安裝 npm。npm 稱為 Node 套件管理員，安裝 Node.js 也會同時安裝 npm。

先用以下指令安裝 nvm: curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash

  • 沒有 curl 要先 sudo apt-get install curl

然後 nvm install --lts 安裝 Node.js 就完成了!

  • 使用 npm --version 確認 npm 是否安裝成功。

我是使用 WSL 安裝，如果想使用 Windows 可以參考以下文件: 在 Windows 上安裝 Node.js

WSL 的指令實際上也是參考 Windows 文件: 在 Windows 子系統 Linux 版 (WSL2) 上安裝 Node.js

安裝 git

Debian: sudo apt install git-all

Windows: Download for Windows

  • 官方載點

Hexo

安裝 Hexo

npm install hexo-cli -g

  • hexo version 檢查是否安裝成功。

初始化 Hexo

  1. hexo init <資料夾名稱>
  2. cd <資料夾名稱>
  3. hexo server

這樣就建立好預設的網頁了。

選擇喜歡的主題

可以去 Hexo Themes 找喜歡的主題，我是使用 Archer，我後面的示範也是用 Archer 來介紹。

  • 建議是挑選還有在維護的主題，比較不會有問題

    喜歡簡潔的也可以選 NexT

挑選好主題後就可以去他們的 github 或是網站看下載方式。

  • hexo-theme-archer

    主題設定的部分可以自己去看他們的 github 研究，不想看我寫的可以直接跳到 Github Pages

安裝 Archer: git clone https://github.com/fi3ework/hexo-theme-archer.git themes/archer --depth=1

  • 主題會放在 themes/archer
  • 必要 dependency: npm install hexo-generator-json-content
    • 沒有這個就沒辦法使用 sidebar index

設定 _config.yml

_config.yml 主要是和 Hexo 有關的設定，themes/archer/_config.yml 則是和主題有關的設定。

  • 改 theme 以及新增 jsonContent
  • 這部分就是一些網站設置，可以自己想要寫什麼
  • author 會影響文章最後 copyright 的 Author
  • language 會影響文章最後 copyright 模板的語言，選中文的話會是簡體中文
    • themes/archer/languages 可以改元素對應的字

設定 /themes/archer/_config.yml

這部分我會介紹我更動過的元素
avatar: 大頭貼，路徑如果類似這樣 /avatar/Misaka.png，要把圖片放到 themes/archer/source/avatar 下面
social: 這裡可以新增各種社交軟體的連結，會以圖片形式在網頁上呈現
SEO_title, SEO_keywords: 不太懂，我設定的和 _config.yml 的 # Site 那裡一樣
main_title: 主畫面的標題
subtitle: 標題下面的字
site_header_image: 主畫面的背景圖，路徑邏輯和大頭貼一樣
post_header_image: 文章內的背景圖
favicon: 瀏覽器分頁上面的那個小小的圖片
copyright: 要不要在文章下面出現作者, 發布日期, license 等等

  • license 可以參考我的寫法

    如果不知道 license 要用哪一個可以去 LICENSE CHOOSER，回答幾個問題他會推薦你適合的 license

Github Pages

建立 Repository

如果沒有 Github 的帳號先註冊一個。

名稱取 username.github.io，然後要設定為 public。

  • example.github.io 為範例，我的話正確是寫 skysoul1024.github.io

Deploy

  1. 先安裝 npm install hexo-deployer-git --save
  2. 修改 _config.yml
  • repo: 點擊 github 頁面的 Code，選 SSH

    不會設定 SSH key 的可以參考 Git 教學和 GitHub 設定指引

  • branch: 現在預設應該是 main?
  1. 最後 這樣就完成布署了!
  • 如果出現 You should configure deployment settings in _config.yml first! 代表 config.yml 有格式錯誤，可能是沒有縮排、元素名稱打錯之類的。

    記得將 _config.yml 的 url 改成 username.github.io
    如果以上設定都沒有問題且成功 deploy，但瀏覽網站沒有正確解析的話可以先等個幾分鐘再去瀏覽。

發表新文章

hexo new [layout] <文章標題>

  • layout: 可以根據 scaffolds 中的模板建立對應的格式。預設 layout 在 _config.yml 的 default_layout 設定。
  • 接著就可以到 source/_posts/ 編輯文章

如果想新增 About Me: hexo new page "about"

  • 就可以在 source/about/index.md 修改自己的 about

    注意: source/about/index.md 中的 layout: about 不可修改

重新 deploy 就可以在自己的網站上看到新文章了。
