Cached File Viewer

題目為一個選單可以選擇 load, read file:

漏洞在 load_file:

  • 中間 if (arena.find(filename) != arena.end()) 如果 arena 中找到一樣的 filename 就會取得 arena 中的內容，且 is_redacted 設定為 false。
  • is_redacted 如果是 true，在 output_content 就會不通過檢查，而印不出 flag:

arena 類似於 cache，在 update_items 最後就會把 load 過的內容放到 arena:

所以只要連續 load 兩次 flag，讓 is_redated = true，就可以繞過 output_content 的檢查。

  • file 的 content 是使用 string_view 去儲存

simple_calc*

source code:

  • check 會檢查 s 中的字元是否為數字以及 s 的長度最多為 5
    • isnumeric() 實際上只要 unicode 字元能擴展成數字就可以通過檢查，例如說 ½ 為二分之一
  • calc 會先檢查 s 中是否包含 + 或 *，如果不包含則是會以 for c in s: x = 10 * x + numeric(c) 這個邏輯計算
    • Example: ⅓1 -> 4 th character is *
  • 最後會印出 text[val]

因此我們只要想辦法找到讓 calc 為 12345679 ~ 12345678 + len(flag)，就可以 leak 所有 flag。

我的隊友最後有靠人腦找出來，而我後來看了別人的 writeup 發現可以使用 z3 解，所以學了一下 z3 的解法。

Exploit

先找出所有 unicode 可能的 value，然後宣告 5 個 Reals 變數，並加入變數範圍的 constraint 以及 x1 * 10000 + x2 * 1000 + x3 * 100 + x4 * 10 + x5 == 12345678:
